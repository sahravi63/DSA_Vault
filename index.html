Aptitude & Verbal Ability

Full-syllabus quick reference: formulas, patterns, tricks, and worked examples for aptitude and verbal reasoning.

Part 1 — Quantitative Aptitude

1. Mixtures & Alligation

  • Adding pure liquid to a mixture: New % = (existing substance qty + added pure qty) ÷ new total volume × 100

  • Replacement method: track water/milk separately — Final conc. = Initial conc. × (1 − removed fraction) per round

  • Ratio target problems: set up water : (milk + x) = target ratio and solve for x

  • Trick: always find the actual quantity of the ingredient first (% × total), not just the percentage

2. Profit & Loss

Concept Formula
Selling Price SP = CP + Profit (or CP − Loss)
Marked Price w/ discount SP = MP × (1 − discount%)
% Profit / Loss (SP − CP)/CP × 100
Two articles, same SP, +p% / −q% CP = SP/(1 ± %) per article; sum CPs vs sum SPs
No gain/no loss, two articles loss% × CP1 = profit% × CP2 → gives CP ratio

Trick: when “sold at same price”, never average the % directly — convert each to CP using SP/(1 ± %), then compare totals.

Quick ratio trick: if loss% on article 1 = profit% on article 2, then CP1 : CP2 = profit% : loss% (inverse of the rates).

3. Time & Work

  • Rate = 1/days. Combined rate = sum of individual rates. Time = 1/(combined rate).

  • Partial work handover: work done by A in t days = t/(A's total days); remaining = 1 − that; time for B = remaining ÷ B's rate

  • Pair-sum trick (A+B, B+C, C+A given): add all three pair rates, divide by 2 to get A+B+C rate → time = 1/(A+B+C)

  • Find one worker alone: third's rate = (A+B+C rate) − (known rates)

4. Speed, Time & Distance

Situation Formula
Basic Speed = Distance / Time
km/h → m/s × 5/18
m/s → km/h × 18/5
Average speed (different speeds) Total Distance ÷ Total Time (not simple average)
Train passing a stationary point/man Time = Train length ÷ Speed
Train crossing a bridge/platform Time = (Train length + Bridge length) ÷ Speed
Two platforms, find train length (L+P1)/T1 = (L+P2)/T2, solve for L
Extra distance if speed increased (New speed − Old speed) × time = extra distance
Mixed foot + vehicle speed x/speed1 + (total − x)/speed2 = total time

Trick: never average two speeds unless time (not distance) is equal on each leg.

5. Simple Interest

SI = (Principal × Rate × Time) / 100

6. Ratio & Average

Average = Sum of values / Number of values

For perimeter/geometry mixed with ratios: use the given formula (e.g., 2(l+b) = Perimeter) and substitute.

7. Number Series — Patterns to Test First

  1. Constant difference (arithmetic): check if consecutive differences are equal

  2. Constant/increasing multiplier (×1.5, ×2.5, ×3.5…): check ratios between terms

  3. Multiply-then-subtract/add pattern (×3 −2, ×3 −2…): check if repeating the operation fits

  4. Increasing difference (+5, +7, +9, +11…): common in difficulty-scaling series

  5. Simple constant addition: differences all equal

Approach: compute the difference row first, then the ratio row if differences aren't constant.

8. Odd One Out — Reasoning

  • Letter pairs: check the alphabetical gap between letters — one pair breaks the common gap

  • Three-letter groups: check if the middle letter = average position of first & last

  • Word groups: classify by category (planet vs satellite, carnivore vs herbivore, tool vs wearable)

9. Cube Problems

Cube of side n, cut into n³ smaller cubes, painted on all outer faces:

Cubes with... Formula n = 4 (64 cubes)
3 faces coloured (corners) Always 8 8
2 faces coloured (edges) 12 × (n−2) 24
1 face coloured (face centers) 6 × (n−2)² 24
0 faces coloured (inner core) (n−2)³ 8

Memorize this table pattern for any n — it's the most reused cube-problem shortcut.

10. Percentages

Value = (Percentage / 100) × Total %Change = (New − Old)/Old × 100

  • Successive % changes: Net% = a + b + (ab/100). E.g. +20 then −10 → 8% net increase

  • Reverse % trick: if A is x% more than B, then B is [x/(100+x)]×100 % less than A

Worked example: A's salary is 25% more than B's → B is less by [25/125]×100 = 20%.

11. Compound Interest (vs Simple Interest)

CI = P(1 + R/100)ⁿ − P For 2 yrs: CI − SI = P × (R/100)²

Half-yearly compounding: use R/2 as rate and 2n as time periods.

Worked example: P=₹10,000, R=10%, T=2 yrs → SI=₹2,000; CI=12,100−10,000=₹2,100; difference=₹100=P×(R/100)².

12. Boats & Streams

Situation Formula
Downstream speed Boat speed + Stream speed
Upstream speed Boat speed − Stream speed
Boat speed in still water (Downstream + Upstream) / 2
Stream speed (Downstream − Upstream) / 2

Worked example: Downstream 14 km/h, upstream 8 km/h → boat speed = 11 km/h, stream = 3 km/h.

13. Pipes & Cisterns

Same logic as Time & Work — outlet pipes count as negative rate.

  • Filling pipe rate = +1/time; emptying/leak pipe rate = −1/time

  • Combined rate = sum of all rates; Time to fill = 1/combined rate

Worked example: A fills in 6 hrs, B empties in 8 hrs, both open: rate = 1/6−1/8 = 1/24 → fills in 24 hrs.

14. Problems on Ages

  • “5 years ago” → (age − 5); “in 10 years” → (age + 10)

  • Set present ages as variables, form 2 equations, solve simultaneously

Worked example: Father = 3× son now; after 5 yrs, father = 2.5× son → son=15, father=45.

15. Calendars

  • Odd days: ordinary year = 1, leap year = 2. 100 yrs = 5, 200 yrs = 3, 300 yrs = 1, 400 yrs = 0

  • Odd days → weekday: 0=Sun, 1=Mon, 2=Tue, 3=Wed, 4=Thu, 5=Fri, 6=Sat

  • Leap year: divisible by 4, but century years must be divisible by 400

16. Clocks

  • Minute hand: 6°/min; Hour hand: 0.5°/min

Angle at H:M = |(30×H) − (5.5×M)| degrees

  • Hands coincide every 65 5/11 minutes

Worked example: Angle at 3:40 = |90 − 220| = 130°.

17. Permutations & Combinations

nPr = n!/(n−r)! (order matters) nCr = n!/[r!(n−r)!] (order doesn't matter)

  • n distinct items in a row: n!. With repeats: n!/(p!q!...)

  • Circular arrangement: (n−1)!, or (n−1)!/2 if clockwise = anticlockwise

Worked example: Select 3 from 6 → 6C3 = 6!/(3!3!) = 20.

18. Probability

P(event) = Favourable outcomes / Total outcomes

  • P(A or B) = P(A) + P(B) − P(A and B) [drop last term if mutually exclusive]

  • P(A and B), independent = P(A) × P(B) P(not A) = 1 − P(A)

Worked example: Card drawn from 52: P(king or queen) = 4/52 + 4/52 = 2/13.

19. Data Interpretation

  • Read the units/scale first — common trap is misreading “in thousands” or “in %”

  • Pie charts: Value = (Sector angle/360) × Total, or (Sector %/100) × Total

  • DI is rarely conceptually hard — it's a speed test; practice fast mental %/ratio calculation

20. Races & Games of Skill

  • “A gives B a start of x m” in a race of d m → A runs d, B runs (d−x) in the same time

  • “A beats B by x m” → when A finishes d, B has covered (d−x)

21. HCF & LCM

HCF × LCM = Product of the two numbers (for exactly 2 numbers)

  • “Largest tile/container size” → HCF. “When events coincide again” → LCM

22. Number System

  • Divisibility: 2 (last digit even), 3 (digit sum ÷3), 4 (last 2 digits ÷4), 5 (ends 0/5), 9 (digit sum ÷9), 11 (alternating sum diff ÷11)

Sum of first n naturals = n(n+1)/2 Sum of squares = n(n+1)(2n+1)/6

23. Partnership

Profit is divided in the ratio of (Investment × Time period) for each partner, not investment alone.

Worked example: A invests ₹5,000×12mo, B invests ₹7,500×8mo → ratio 60000:60000 = 1:1 → equal split.

24. Basic Algebra / Quadratic Equations

x = [−b ± √(b² − 4ac)] / 2a for ax² + bx + c = 0

  • Sum of roots = −b/a; Product of roots = c/a — useful for fast MCQ elimination

25. Mensuration (Areas & Volumes)

Grammar Rules (High-Yield)

  • Subject-verb agreement: singular subject → singular verb (she goes); “Each” always takes singular verb (each... has)

  • Modal + base verb: after did/does/can/should, use base form (did not know, not knew)

  • Tag questions: positive statement → negative tag (isn't it?); negative → positive tag

  • Indirect speech: present continuous → past continuous (“I am eating” → he said he was eating)

  • Passive voice: match tense — past simple active → was/were + past participle (built → was built)

  • Future perfect: “By [future time], I ___” → will have + past participle

  • Prepositions: between = two people/things; among = more than two

  • Fixed pairs: No sooner...than; So that (purpose); good at (not “good good”)

  • “It is high time” → followed by past tense, even though it refers to now

  • Articles (a/an/the): “a” before consonant sounds, “an” before vowel sounds (an hour, a university — sound, not spelling); “the” for specific/already-known nouns

  • Determiners: this/that/these/those (proximity), some/any (some in positive, any in negative/question), much (uncountable), many/few (countable), little (uncountable, almost none)

  • Pronoun agreement: pronoun must match its antecedent in number and gender — “Each student submitted their/his or her assignment” not “their” if strict singular is tested

  • Pronoun case: subject pronouns (I, he, she, they) do the action; object pronouns (me, him, her, them) receive it — “between you and me,” never “between you and I”

  • Parallelism: items joined by and/or/but must share the same grammatical form — “She likes reading, writing, and to swim” is wrong; should be “reading, writing, and swimming”

  • Modifiers: keep modifiers next to the word they describe — a misplaced modifier changes meaning (“She almost drove her kids to school every day” vs “She drove her kids to school almost every day”)

  • Conditional sentences: Zero (if + present, present — general truth); First (if + present, will + base — real future); Second (if + past, would + base — unreal present); Third (if + past perfect, would have + past participle — unreal past)

Spelling

  • Common trap word: Exaggerate (double “gg”, single “r”)

  • Sound out syllable by syllable; watch for double consonants and silent letters

Sentence Completion (One-Word, No Options)

  • Must be spelled and grammatically perfect — no partial credit

  • Look for fixed collocations (get rid of, good at, extricate from) rather than literal translation

Error Identification / Correction Strategy

  1. Scan for subject-verb agreement first — most common error type

  2. Check tense consistency across the sentence

  3. Check preposition and article usage

  4. If nothing seems wrong, consider “No Error” — only after checking the above

Passage/Sentence Ordering (Para Jumbles)

  1. Find the opening sentence — no backward-referring pronoun (avoid This/It/He unless subject already given)

  2. Find linking words (however, therefore, thus, but, also, meanwhile)

  3. Look for pronoun chains — a sentence using he/it/this follows the sentence that introduced that noun

  4. Find the concluding sentence — summarizing or result tone

  5. Build pairs first, then arrange pairs into the full sequence

Cloze Test

  • Read the whole passage once before filling any blank

  • Focus on tense consistency, preposition collocations, connector logic (contrast/cause/addition)

  • Eliminate grammatically correct but contextually odd options

Reading Comprehension

  • Skim first for structure — main idea, tone, each paragraph's role

  • Read questions, then go back to the passage to locate answers

  • Inference questions: answer must be supported by the passage, not outside knowledge

  • Tone/attitude questions: look for adjectives and connectors (however, surprisingly, unfortunately)

Part 3 — Logical Reasoning — High-Yield Section

Many companies (TCS, Infosys, Capgemini, Accenture, Wipro) ask 15–30 Logical Reasoning questions — this is consistently one of the highest-weightage sections. Master the topics below in order of frequency.

1. Blood Relations

  • Approach: draw a family tree as you read; use + for male, − for female, and mark generations as separate rows so relations are visually obvious

  • Chain trick: for long chains (“A is B's father's mother's son”), resolve one relation at a time from left to right, redrawing the tree after each step

  • Watch for: gender-ambiguous terms (cousin, sibling, child, parent) — the puzzle may deliberately withhold gender to test careful reading

2. Direction Sense

  • Standard orientation: facing North — right hand points East, left hand points West (unless the question states “facing south” etc.)

  • Left/Right turns: facing North → turn right = face East; turn left = face West; a 180° turn always reverses direction

  • Net displacement: plot each move on an X-Y grid (N/S = Y-axis, E/W = X-axis); final distance from start = √(net-X² + net-Y²) using Pythagoras

  • Shadow trick: at sunrise, shadow falls West; at sunset, shadow falls East (sun rises in the East)

  • Common trap: “turned towards left” vs “turned left” can mean the same thing — read the exact wording; some papers use “clockwise/anticlockwise” instead of left/right

3. Seating Arrangement — Linear & Circular

  • Linear — facing same direction: left/right are from each person's own perspective, not the reader's, unless stated “facing the reader”

  • Linear — facing reader (North-facing row vs South-facing row): left/right flip between two rows facing each other — the person directly opposite has left and right reversed

  • Circular — facing center: clockwise = to the right of a given person; anticlockwise = to the left

  • Circular — facing outward (away from center): left/right are reversed compared to facing-center arrangements — always check which way the group faces first

  • Fixed-point trick: start placing people from the person with the most constraints (e.g. “immediate neighbours of X”) rather than reading top to bottom

  • Elimination trick: draw all valid slots first, then cross out positions violating each clue one at a time — don't try to satisfy all clues in one pass

4. Coding-Decoding

Rule Type How it works Example
Alphabet Position Each letter shifted by a fixed number of positions forward/backward CAT → DBU (each letter +1)
Reverse Alphabet Letter replaced by its mirror position from the other end (A↔︎Z, B↔︎Y...) A→Z, B→Y, C→X (position + mirror position = 27)
Symbol Coding Letters/words mapped to arbitrary symbols via a given key; decode using the key only If ‘@’ = A, ‘#’ = B, then A B → @ #
Mixed Coding Combination of position shift AND reversal, or letters + numbers together Word reversed, then each letter shifted — solve in two steps
Number Coding Each letter assigned a number (often A=1...Z=26, or a custom key) WORD → 23-15-18-4 (standard A1Z26)
Letter Coding Whole words substituted with code words via a given key table If ‘rain is wet’ = ‘pit sil kot’, decode word-by-word using the key

  • Solving method: always find the transformation rule using the given example pair first, then apply the exact same rule to the target word — don't assume a standard shift without checking the example

  • Double-check: verify your rule against every letter in the example, not just the first one — many patterns use different shifts for vowels vs consonants

5. Analogy & Classification

  • Analogy — find the relationship first: cause-effect, part-whole, synonym, antonym, degree, or function — e.g. Doctor:Hospital :: Teacher:School (place of work)

  • Analogy trap: the relationship must hold in the same direction and order as the original pair — not just any related pair

  • Classification (Odd One Out): group items by category, not surface similarity — e.g. among Lion, Tiger, Whale, Leopard: Whale is odd (not a land carnivore/cat family)

  • Numeric classification: check prime vs composite, odd vs even, perfect square, or digit-sum patterns when the odd-one-out is a set of numbers

6. Statement & Conclusion / Assumption / Syllogism

  • Statement & Conclusion: a conclusion follows only if it is directly and necessarily supported by the statement — don't use outside knowledge or “likely” inferences

  • Statement & Assumption: an assumption is something implicitly taken for granted for the statement to make sense — test it by negating it: if the statement falls apart, it was an assumption

  • Syllogism — use Venn circles: draw a circle for each term; “All A are B” = A circle fully inside B; “Some A are B” = circles overlap; “No A are B” = circles fully separate

  • Syllogism — particular vs universal: a conclusion using “all/no” needs both premises to be universal; if either premise is “some”, the conclusion can only be “some” at best

  • Syllogism trap: “Some A are B” does NOT imply “Some A are not B” — both can be independently true or false; check each conclusion against every valid Venn diagram, not just one

  • Trick: always fill in the innermost region (A∩B∩C) first when given raw data, then work outward — solving in that order avoids double-counting

8. Ranking & Ordering

  • Rank from one end: if X is Nth from the top in a group of T, X's rank from the bottom = T − N + 1

  • Total people in a row: if X is Nth from the left and Mth from the right, total = N + M − 1

  • Between two people: if X is Nth and Y is Mth from the same end, people strictly between them = |N − M| − 1

  • Rank change: “moves up by k places” means new rank (from top) = old rank − k; “moves down” means old rank + k

  • Century codes: 1600s/2000s = 6, 1700s = 4, 1800s = 2, 1900s = 0 (repeats every 400 years)

  • Day calculation shortcut (Zeller-style): Day code = (date + month code + last-2-digits-of-year + last-2-digits÷4 + century code) mod 7, where result 0=Sunday, 1=Monday...6=Saturday

  • Faster mental route: memorize the weekday of one “anchor date” per year (e.g. 1st Jan or the last day of Feb) and count forward using odd-day jumps (1 odd day per common year, 2 per leap year)

Worked example: 15 Aug 1947 — date=15, Aug code=2, yy=47, yy÷4=11, century(1900s)=0 → (15+2+47+11+0) mod 7 = 75 mod 7 = 5 → Friday.

10. Clock Reasoning

Core angle formula and coincidence timing are covered in Part 1, Section 16 — additional reasoning-style shortcuts below.

  • Mirror image time: reflect the time about 6:00 or 12:00 — formula: mirror time = 11:60 − given time (e.g. 4:20 → 7:40)

  • Water image time: reflect about the vertical (9:00–3:00) axis — formula: water image = 8:60 − given time for a 12-hour dial reflected top-bottom... in practice, use 12:00 minus elapsed time from 6:00 for quick checks

  • Gaining/losing clocks: if a clock gains/loses m minutes every h hours, actual elapsed time vs shown time ratio = (h×60) : (h×60 ± m) — use this ratio to scale any duration

  • Hands at right angle: hands are 90° apart 22 times in 24 hours (twice roughly every 65 5/11 minutes cycle, skipping one occurrence near 3 and 9 o'clock)

11. Input–Output Reasoning

  • Identify the rule first: compare Step 0 (input) to Step 1 — look for sorting (alphabetical/numerical), shifting to ends, or arithmetic operations on numbers

  • Common patterns: words sorted alphabetically one-by-one shifted to an end each step; numbers sorted by value; numbers replaced by their squares/products; word-number pairs swapped

  • Track two things separately: if both words and numbers are present, solve their movement rules independently — they rarely interact

  • Verify on a second step: confirm your rule correctly predicts Step 2 from Step 1 before applying it to answer the question — a rule that only fits one step transition may be wrong

12. Puzzle Sets (Multi-condition puzzles)

  • Types: floor puzzles, box/stack puzzles, scheduling (day/month) puzzles, comparison (age/height/salary) puzzles, and combined seating + attribute puzzles

  • Strategy: list all entities and all attributes in a grid before reading conditions in detail; fill in directly-stated facts first, then apply elimination for indirect clues

  • Comparison puzzles: convert “taller than / older than / earns more than” clues into a single ranked order chain, merging partial chains as you go

  • Floor/box puzzles: number floors bottom-to-top (or boxes top-to-bottom) exactly as the question defines — a reversed numbering assumption is the most common error

  • Time management: if a puzzle set has 4–5 questions attached, spend up to 3–4 minutes solving the puzzle itself — the setup cost is repaid across all its questions

Data Sufficiency

Frequently asked in Accenture, Capgemini, and Infosys. Tests whether given statements provide enough information to answer the question — not whether you can find the actual answer.

Answer Choice Meaning
Statement I alone is sufficient Question can be answered using only Statement I, but Statement II alone is not enough
Statement II alone is sufficient Question can be answered using only Statement II, but Statement I alone is not enough
Both together are sufficient Neither statement alone is enough, but combining both answers the question
Either statement alone is sufficient Statement I alone works AND Statement II alone also works (independently)
Neither is sufficient Even combining both statements does not answer the question

  • Golden rule: evaluate Statement I completely on its own first, forgetting Statement II exists; then evaluate Statement II alone; only combine them if neither works individually

  • Sufficiency ≠ a nice answer: a statement is “sufficient” if it gives exactly one definite answer (including “no”) — you don't need to compute the final number, just confirm it's uniquely determined

  • Common trap: a statement that lets you calculate two possible values (e.g. x = 4 or x = −4) is NOT sufficient, even though it narrows things down

  • Don't import outside assumptions: e.g. don't assume a variable is positive or an integer unless the question or a statement says so
